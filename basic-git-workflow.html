Basic Git Workflow

Workflow based on your own local projects.

From your projects directory:

  • git init - initializes a new repo in git.
  • git add . = add all changes or, git add filename.ext = add a specific file.
  • git commit -m "message goes here!"
  • git push origin (branch name). { This only works once you've created a repo on github. }

Workflow on exsiting project.

From your projects directory:

  • git status - to see what changes have been made.
  • git add . = add all changes or, git add filename.ext = add a specific file.
  • git commit -m "message goes here!"
  • git push origin (branch name).

If working with other people on a project you should do a git pull
before you push any code to github. That way you have the most
updated version.

Forking and Cloning

Forking copies a project on github and puts the code base in "your repositories".
You can then hit the clone/download button to do one or the other. This is to get the source code on your local machine.

  • Download - will simply download a zip file.
  • Clone - In the terminal either create a directory or be in a directory you
    want project in and type git clone (the address). The address of the
    project on github will appear when you hit the clone/download button, from there you can
    copy it to your clipboard and paste it in the terminal.
